# Step 1.1: Translate Requirements into Specific Questions

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

Analyze the requirement description sentence by sentence. Extract:
- Every business concept and verb
- For each concept, ask: what class/table/field/enum does it correspond to in code?
- For each verb, ask: where is the entry point? What processing does it go through? What data does it ultimately affect?
- List all questions, mark each as "known" or "to investigate"

## Checklist

Use these items in your **[CHECK]** stage:

- Every business noun → "what is it in code?" question
- Every business verb → "where is entry point?" question
- At least 3 implicit assumptions identified
- All questions marked "known" or "to investigate"
- "To investigate" items have clear exploration direction

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: What breaks this? → If I missed a concept, what happens?
- Q2: What didn't I check? → Name 1 potentially missed implicit requirement
- Q3: Expert criticism? → Would analyst think I understood deeply?

## Minimum Loops

You must complete at least **2 loops** before you can PASS this step.
