# Step 3.1: Database Model Design

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

Design complete CREATE TABLE DDL strictly following [data-integrity spec](../../../specs/data-integrity/spec.md):
- All fields, indexes, comments
- 9 audit fields in standard order
- Primary key: BIGINT AUTO_INCREMENT
- Indexes ≤ 5, using `idx_`/`uniq_` prefix
- No foreign keys, triggers, stored procedures, or views
- Every field must have COMMENT

## Checklist

Use these items in your **[CHECK]** stage:

- All fields support E2E Flow data operations
- Indexes cover all query scenarios
- Strictly follows database-design-common spec
- 9 audit fields in standard order
- Every field has COMMENT
- Primary key is BIGINT AUTO_INCREMENT

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: What breaks this? → If E2E Flow has a query operation, do my indexes support it?
- Q2: What didn't I check? → Is there a field I "think is unnecessary" but should actually add?
- Q3: Expert criticism? → If DBA reviews this DDL, what optimization would they suggest?

## Minimum Loops

You must complete at least **2 loops** before you can PASS this step.
