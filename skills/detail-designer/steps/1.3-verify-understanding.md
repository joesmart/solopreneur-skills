# Step 1.3: Verify Understanding Depth

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

Apply two verification standards:

**Standard 1 — Can you fully explain the data flow in your own words?**
Where does data enter, what processing does it go through, where is it stored / what is returned?
If you cannot explain this clearly → loop back to 1.2.

**Standard 2 — Can you predict the "pitfalls" of this requirement?**
Example: "This change will affect module XXX's YYY logic, needs attention."
If you cannot predict pitfalls → understanding is not deep enough → loop back to 1.2.

If there are points that cannot be determined, **ask the user for clarification** with specific questions.

## Checklist

Use these items in your **[CHECK]** stage:

- Can explain complete data flow in own words (without looking at notes)
- Can predict at least 2 potential pitfalls or risk points
- Can answer "why this design" not just "what it is"
- If asked to draw architecture diagram now, could do it
- All uncertain points clarified with user (if any)

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: What breaks this? → If I start coding now, where would I get stuck?
- Q2: What didn't I check? → Name 1 area where my understanding is shallow
- Q3: Expert criticism? → If senior engineer reviews my understanding, what would they point out?

## Minimum Loops

You must complete at least **1 loop** before you can PASS this step.

## Final Output

Record final requirement understanding conclusion and Reference Module.
