# Step 3.5: Generate Final Document

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

Load [references/output-template.md](../references/output-template.md) and fill every section with real design data from Phase 1-3.

**Hard constraints on output:**
- **NO** placeholders or template examples — all content must be real design data
- **NO** omitting any template section — every section must exist and be complete
- **NO** reordering sections — section order must match template exactly
- **NO** including the data walkthrough (3.3) in the output — it stays internal

## Checklist

Use these items in your **[CHECK]** stage:

- All template sections filled with real design data (no placeholders)
- No template section omitted
- Section order matches template exactly
- Data walkthrough (3.3) NOT included in output
- Document is "coding-ready" (developer can start immediately)

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: What breaks this? → If I randomly check a section, would I find template examples or TODOs?
- Q2: What didn't I check? → If developer says "this doc isn't detailed enough", which part would it be?
- Q3: Expert criticism? → If technical director reviews this doc, would they consider it "production-ready"?

## Minimum Loops

You must complete at least **2 loops** before you can PASS this step.

## Final Output

After completing this step, you MUST:

1. **Create the designs directory** if it doesn't exist at the project root
2. **Write the design document** to `designs/design-[feature-name]-[timestamp].md`
   - Use kebab-case for feature name (e.g., `user-authentication`, `order-payment`)
   - Use timestamp format: `YYYYMMDD-HHmmss` (e.g., `20260302-143022`)
   - Example: `designs/design-user-authentication-20260302-143022.md`
3. **Verify the file was created** by reading it back to confirm content integrity

**This is a mandatory step — the design document MUST be persisted to the designs folder. Do not skip this step under any circumstances.**
