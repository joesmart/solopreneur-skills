# Step 2.2: Validate Against 10 Hard Constraints

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

For each candidate solution from Step 2.1, check ALL 10 constraints. **Every constraint must pass — no exceptions.**

| # | Constraint | How to Check |
|---|-----------|--------------|
| 1 | Minimal code change | Any unnecessary files/classes/methods? Can anything be removed? |
| 2 | No over-engineering | Simple requirement getting an overly complex solution? |
| 3 | Root cause solution | Addressing the fundamental problem or surface symptoms? |
| 4 | No band-aids | Is logic in the right place? Or patching at the surface? |
| 5 | Extensible | Can this solution continue to evolve with future requirements? |
| 6 | Performant & stable | SQL indexed? N+1 issues? Large loops? |
| 7 | Minimal blast radius | Modified shared methods? Will other callers break? |
| 8 | Best practices | Existing framework/tool/pattern available? |
| 9 | Architecture consistency | Consistent with the Reference Module's architecture? |
| 10 | Data integrity | Concurrent writes cause dirty data? Need transactions? Safe on retry? |

For each failing candidate, state: "Fails constraint #X because [reason]."
Select the best candidate and explain why others were eliminated.

## Checklist

Use these items in your **[CHECK]** stage:

- All 10 constraints checked for each candidate (not skipped any)
- Each constraint check has concrete evidence (not "should be fine")
- Failed candidates clearly state which constraint failed and why
- Best candidate passes ALL 10 constraints
- Can explain why other candidates were eliminated

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: What breaks this? → Did I "rubber-stamp" any constraint because I like this solution?
- Q2: What didn't I check? → If I re-check constraints now, would I change any evaluation?
- Q3: Expert criticism? → If Tech Lead reviews my constraint validation, would they think I was rigorous enough?

## Minimum Loops

You must complete at least **1 loop** before you can PASS this step.

## Final Output

Record selected solution and elimination reasons.
