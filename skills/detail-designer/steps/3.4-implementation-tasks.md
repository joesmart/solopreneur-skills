# Step 3.4: Implementation Todo List

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

Convert every node in the E2E flow into executable coding tasks at class→method granularity.

**MANDATORY**: Load [references/output-template.md](../references/output-template.md) and follow the "Implementation Tasks Format" section exactly.

List tasks in layer order: Database → Entity → Mapper → Service → BizService → Controller → DTO/VO → Enum/Config

Each task must include:
- Status emoji: 🔲 TODO / 🔄 IN PROGRESS / ✅ DONE
- Related Specs references
- Done When checklist with specific, verifiable criteria
- Implementation Details: File, Class, Method signature, Responsibility, Steps
- Clear separation with `---` between tasks

## Checklist

Use these items in your **[CHECK]** stage:

- Every E2E Flow node has corresponding task
- Every task has Done When checklist with specific, verifiable criteria
- Tasks in correct layer order (Database → Entity → Mapper → Service → BizService → Controller → DTO/VO → Enum/Config)
- Includes auxiliary classes (DTO/VO/Enum/Config)
- Every task references related Specs
- Status emoji used (🔲 TODO / 🔄 IN PROGRESS / ✅ DONE)
- Implementation Details section complete (File/Class/Method/Responsibility/Steps)
- Tasks separated with `---` for readability

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: What breaks this? → If developer follows this task list, would they discover missing files?
- Q2: What didn't I check? → Is any task's Done When criteria too vague?
- Q3: Expert criticism? → If project manager reviews this task list, would they think it's executable?

## Minimum Loops

You must complete at least **2 loops** before you can PASS this step.
