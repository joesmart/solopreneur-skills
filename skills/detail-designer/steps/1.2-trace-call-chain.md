# Step 1.2: Trace Code Along the Call Chain

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

Starting from the entry point (Controller/API/scheduled task), follow the call chain layer by layer:

```
Controller → BizService → Service → Mapper → DB
```

For each key class/method, record:
- File path, method signature, core logic, data transformations

**Find the most similar existing feature module as a Reference Module.** Read the reference module's code to understand its layered architecture, naming conventions, and code style.

Each answered question may raise new questions — if so, continue exploring.

## Checklist

Use these items in your **[CHECK]** stage:

- Complete call chain traced (Controller → BizService → Service → Mapper → DB)
- Each layer: file path + method signature + core logic documented
- Reference Module identified with similar architecture
- At least 1 cross-cutting concern explored (cache/transaction/permission/exception)
- All Phase 1.1 questions answered with code evidence

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: What breaks this? → If there's a hidden layer (cache/proxy), does it break my understanding?
- Q2: What didn't I check? → Name 1 aspect I didn't explore (and why it's OK)
- Q3: Expert criticism? → Would architect think this is thorough enough?

## Minimum Loops

You must complete at least **2 loops** before you can PASS this step.
