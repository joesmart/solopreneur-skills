# Step 3.3: Data Walkthrough Verification (thinking only, NOT in output)

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

Construct at least 2 sets of concrete business data (normal scenario + exception scenario). Trace data through each node in the flow, verifying input/output connections.

**This walkthrough is for internal verification only — it does NOT appear in the final design document.**

## Checklist

Use these items in your **[CHECK]** stage:

- At least 2 concrete data sets constructed (normal + exception)
- Data traced through every node with specific values (not "some ID")
- Every node's input/output transformation verified
- At least 1 design flaw or inconsistency discovered
- All discovered issues fixed in Phase 3.2

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: What breaks this? → Is my data specific enough (concrete values vs "some user ID")?
- Q2: What didn't I check? → Is there a node's data transformation I "assumed works" without verifying?
- Q3: Expert criticism? → If QA uses my data to test, would they find bugs I missed?

## Minimum Loops

You must complete at least **2 loops** before you can PASS this step.
