# Thinking Discipline

After each major step, STOP and reflect:

1. **Evidence check**: Is every conclusion backed by code evidence? No guessing allowed.
2. **Completeness**: Did I cover all branches, edge cases, and dependencies?
3. **Challenge**: What if my interpretation is wrong? What did I miss?
4. **Confidence**: Am I confident enough to proceed, or should I dig deeper?

If any answer is NO → dig deeper before proceeding.
If all YES → move to next step.

## When to Apply

- After locating suspected root cause (bugfix)
- After designing a solution (designer)
- After implementing code (implementer)
- After each test case trace (verifier)
- After completing each review dimension (reviewer)
