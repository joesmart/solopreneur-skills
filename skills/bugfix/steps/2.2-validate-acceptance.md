# Step 2.2: Validate Against 10 Acceptance Criteria

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

For each candidate fix from Step 2.1, check ALL 10 acceptance criteria from SKILL.md:

1. Minimal code change
2. No over-engineering
3. Root cause fix
4. No band-aids
5. Extensible
6. Performant & stable
7. Minimal blast radius
8. Best practices
9. Architecture consistency
10. Data integrity

**Every criterion must pass — no exceptions.**

For each failing candidate, state: "Fails criterion #X because [reason]."
Select the best candidate and explain why others were eliminated.

## Checklist

Use these items in your **[CHECK]** stage:

- All 10 criteria checked for each candidate
- Each criterion marked ✓ or ✗ with evidence
- Failing candidates eliminated with clear reasons
- Best candidate selected with justification
- Selected candidate passes all 10 criteria

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: Observation 异常？ → Did I rubber-stamp any criteria without genuine verification?
- Q2: 解读正确？ → Am I being honest about trade-offs, or rationalizing a preferred solution?
- Q3: 还有遗漏？ → If ALL candidates fail, should I loop back to 2.1?

## Minimum Loops

You must complete at least **2 loops** before you can PASS this step.
