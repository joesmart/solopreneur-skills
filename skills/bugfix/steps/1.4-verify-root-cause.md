# Step 1.4: Verify Root Cause Certainty

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

Apply two verification standards to ensure 100% certainty:

**Standard 1 — Can you describe the root cause in one precise sentence?**
Format: "In file XXX, method YYY, line ZZZ, when condition AAA is true, variable BBB is CCC, causing DDD exception."

**Standard 2 — Can you explain why it didn't fail before but fails now?** (if applicable)
Example: "Previously all data had default values; this time null was passed in."

## Checklist

Use these items in your **[CHECK]** stage:

- Root cause described in one precise sentence with file/method/line/condition/variable/error
- Can explain timing: why it fails now but not before (if applicable)
- Root cause matches the ❌ step from Phase 1.3
- No vague terms like "might be", "possibly", "could be"
- 100% confident this is the actual root cause

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: Observation 异常？ → Is there any ambiguity in my root cause statement?
- Q2: 解读正确？ → Am I confusing symptom with root cause?
- Q3: 还有遗漏？ → Can I explain the timing and conditions with absolute precision?

## Minimum Loops

You must complete at least **2 loops** before you can PASS this step.
