# Step 2.1: Brainstorm Candidate Fixes

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

List at least 2 candidate fixes. For each:
- One-sentence summary of core approach
- Which files and methods to modify
- What the key change is

Do not evaluate yet — list all possibilities first.

## Checklist

Use these items in your **[CHECK]** stage:

- At least 2 candidate fixes generated
- Fixes are genuinely different in approach (not just variations)
- Each fix specifies files/methods to modify
- Each fix describes the key change
- Have not started evaluation yet (pure brainstorming)

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: Observation 异常？ → Are all my fixes based on the same assumption?
- Q2: 解读正确？ → Am I fixing the root cause or just adding a band-aid?
- Q3: 还有遗漏？ → Is there a simpler or more architecturally correct approach?

## Minimum Loops

You must complete at least **2 loops** before you can PASS this step.
