# Step 1.1: Translate Bug Symptoms into Specific Questions

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

Analyze the bug description sentence by sentence. Extract:
- Error messages, trigger conditions, expected vs actual behavior
- For each piece of information, ask: which class/method/branch does this correspond to?
- List all questions, mark each as "known" or "to investigate"

If the bug description is unclear, **ask the user for clarification** with specific questions.

## Checklist

Use these items in your **[CHECK]** stage:

- Every error message → "where is it thrown?" question
- Every trigger condition → "which code path?" question
- Expected vs actual behavior clearly documented
- At least 3 implicit assumptions identified
- All questions marked "known" or "to investigate"
- "To investigate" items have clear exploration direction

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: Observation 异常？ → Are there contradictions in the bug description?
- Q2: 解读正确？ → Did I misinterpret any symptoms?
- Q3: 还有遗漏？ → What implicit conditions might I have missed?

## Minimum Loops

You must complete at least **2 loops** before you can PASS this step.
