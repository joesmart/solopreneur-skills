# Step 2.4: Output Fix Plan Summary

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

Present the final fix plan for human review using the exact format from SKILL.md:

```
══════════════════════════════════════════════════
Bug Fix Plan (待人工审查)
══════════════════════════════════════════════════

Root Cause:
  [一句话精确描述根因]

Selected Fix:
  [一句话概括修复方案]

Changes Required:
  1. [文件路径] → [方法名]: [具体改动描述]
  2. [文件路径] → [方法名]: [具体改动描述]

Acceptance Criteria: 10/10 passed

Risk Assessment:
  - Blast radius: [影响范围评估]
  - Edge cases verified: [已验证的边界场景]
  - Remaining risks: [如有残余风险列出,否则 "None"]

⚠️ 请人工审查以上方案后再执行修复
```

## Checklist

Use these items in your **[CHECK]** stage:

- Fix plan uses exact format from SKILL.md
- Root cause is one precise sentence
- Selected fix is clearly summarized
- All changes listed with file/method/description
- All 10 acceptance criteria confirmed passed
- Risk assessment includes blast radius, verified edge cases, remaining risks
- Human review reminder included

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: Observation 异常？ → Is the fix plan clear enough for a human reviewer?
- Q2: 解读正确？ → Does it contain all necessary information to implement?
- Q3: 还有遗漏？ → Any ambiguity that should be clarified?
