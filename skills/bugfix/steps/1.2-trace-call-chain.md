# Step 1.2: Trace Code Along the Call Chain

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

This step uses a **three-stage progressive exploration** approach to ensure complete understanding of bug-related code. Execute all three stages within your ReAct loops:

### Stage A: Locate Error Site

**Goal:** Find the exact location where the bug occurs and identify the immediate cause.

**What to do:**
1. Based on Step 1.1 clues, locate the file and line number where the error occurs
2. Read the file and understand the logic of the error line
3. Identify all variables and method calls involved in the error line
4. Record the direct cause of the error (e.g., null.toUpperCase(), array[-1], divide by zero)

**Output:** ERROR SITE card with fields: File, Method, Line, Code, Error, Cause, Variables

### Stage B: Data Origin Tracing

**Goal:** For each key variable involved in the bug, trace back to the true origin of the data.

**What to do:**

1. **Immediate source**
   - Where does the variable come from? (object.method(), parameter, local calculation)
   - Are there any intermediate transformations?

2. **Object source**
   - Where does the object come from? (method parameter, service call, cache)
   - What is the complete call chain?
   - Trace: Controller → BizService → Service → Mapper → DB

3. **Data ultimate origin**
   - Where is the data stored? (database table, external API, config file, cache)
   - What is the schema/constraint? (nullable? default value? data type?)
   - Under what scenarios can abnormal values occur? (boundary conditions, concurrency, legacy data)

4. **Data producers**
   - Who is responsible for writing this data? (find all INSERT/UPDATE code)
   - Is there validation during write? (null check, format check, range check)
   - Has there been any historical changes? (recent code changes, data migration)

**Output:** DATA ORIGIN TRACE card with fields: Variable, Trace Path, Data Schema, Data Producers, Abnormal Value Scenario

### Stage C: Impact Domain Analysis

**Goal:** Find all affected code paths and related logic.

**What to do:**

1. **Downstream tracing**
   - Where else is the key variable used in the current method?
   - Where else is the key variable used in other methods?
   - Is there null checking at each usage point?

2. **Lateral detection**
   - What other methods are called within the current method?
   - Do these methods also depend on the same data?
   - Are there hidden side effects? (database writes, external API calls, cache updates)

3. **Conditional branches**
   - What if/switch branches exist?
   - What are the trigger conditions for each branch?
   - Which branch does the bug occur in?

4. **Exception handling**
   - Is there try-catch?
   - How is the exception handled? (logged, rethrown, swallowed)
   - What error does the end user see?

**Output:** IMPACT DOMAIN MAP card with fields: Variable Usage, Related Logic, Conditional Branches, Exception Handling

## Execution Strategy

**CRITICAL:** You must complete all three stages (A, B, C) within your ReAct loops. You CANNOT proceed to Step 1.3 until all 15 checklist items pass.

**Recommended execution pattern:**

- **Loop 1:** Focus on Stage A (locate error site) + partial Stage B (immediate source)
  - Expected pass rate: 5-8/15
  - If pass rate < 5, continue looping on Stage A

- **Loop 2:** Complete Stage B (trace to ultimate origin and find producers)
  - Expected pass rate: 10-12/15
  - If pass rate < 10, continue looping on Stage B

- **Loop 3:** Complete Stage C (impact domain analysis)
  - Expected pass rate: 15/15
  - If pass rate < 15, continue looping on Stage C

You may need more loops if the codebase is complex. Continue looping until all checklist items pass.

## Checklist

Use these items in your **[CHECK]** stage:

**CRITICAL:** You must pass ALL items below (15/15) to reach PASS state. The grouping is for organization only - you cannot PASS by completing just one stage.

**Stage A - Error Site:**
- ✓ Error file, method, and line number located
- ✓ Error line code and direct cause identified
- ✓ All variables involved in error line documented

**Stage B - Data Origin:**
- ✓ Each key variable's immediate source identified
- ✓ Call chain traced to database/external API/config
- ✓ Data schema/constraints confirmed (nullable? type? default?)
- ✓ Key data producers (write code) found and validated
- ✓ Abnormal value scenarios identified (when/why null occurs)

**Stage C - Impact Domain:**
- ✓ All usage points of key variables identified
- ✓ Null checking status at each usage point confirmed
- ✓ Related logic within method explored
- ✓ All conditional branches identified
- ✓ Exception handling path traced

**Overall:**
- ✓ All Phase 1.1 questions answered with code evidence

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: Observation 异常？
  → Did I find any unexpected code paths or hidden layers?
  → **Does the data origin match expectations?**
  → **Did I find any data producers without validation?**

- Q2: 解读正确？
  → Did I correctly interpret the call chain and data flow?
  → **Did I trace data origin to the true source (not stopping at intermediate layers)?**
  → **Did I correctly identify which branch the bug occurs in?**

- Q3: 还有遗漏？
  → Are there parallel execution paths or async operations I missed?
  → **Are there other usage points of key variables I haven't explored?**
  → **Are there side effects in related methods I haven't detected?**
  → **Are there other data producers I haven't found?**

## Minimum Loops

You must complete at least **3 loops** before you can PASS this step.

- Loop 1 should cover Stage A + partial Stage B
- Loop 2 should complete Stage B
- Loop 3 should complete Stage C

If the codebase is complex, you may need 4-5 loops. Continue until all checklist items pass.

## Output Summary

After completing all three stages, you should have collected:

1. **Trigger Data** - Abnormal value scenario from Stage B for Step 1.3 test data
2. **Execution Path** - Branch analysis from Stage C for Step 1.3 trace path
3. **Related Logic** - Method calls from Stage C for Step 1.3 execution flow
4. **Root Cause Hypothesis** - Combined analysis from all stages
