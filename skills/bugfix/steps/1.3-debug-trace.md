# Step 1.3: Step-by-Step Debug Trace

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

Like a debugger stepping through code, trace execution step by step with concrete data. Follow the **Debug Trace Protocol** from SKILL.md:

1. **Declare Trigger Data** - Define all context data that reproduces the bug
2. **Step-by-Step Trace** - Draw STEP boxes with all five fields (Input, Execute, Output, Expect/Actual/Verdict)
3. **Root Cause Card** - Summarize the precise root cause

Every step is a complete `Input → Execute → Output → Verdict` closed loop.

## Checklist

Use these items in your **[CHECK]** stage:

- Trigger data declared with concrete values (no vague "some value")
- Every STEP has all five fields (Input, Execute, Output, Expect, Actual, Verdict)
- Every STEP has verdict line with ✅ or ❌
- All ❌ steps have `★ Root Cause:` annotation
- Branches evaluated as separate STEPs
- STEPs connected with ▼ arrows
- Root Cause Card completed with file/method/line/condition/error

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: Observation 异常？ → Are all variable values concrete? Any vague placeholders?
- Q2: 解读正确？ → Did I correctly evaluate each step's expected vs actual behavior?
- Q3: 还有遗漏？ → Are there hidden execution paths I didn't trace?
