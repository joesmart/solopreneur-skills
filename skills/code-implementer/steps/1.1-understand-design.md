# Step 1.1: Understand Design Document

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

Deep dive into the design document or requirements:
- Extract all implementation points from the user's input (design doc, requirements, verbal instructions)
- Read all referenced existing files to understand context and patterns
- Identify reference modules to follow their code style
- Mark any ambiguities or unclear points for clarification
- Do not make assumptions without code evidence

## Checklist

Use these items in your **[CHECK]** stage:

- All implementation requirements extracted from user input
- All referenced existing files read and understood
- Reference modules identified and style understood
- All ambiguities marked or clarified
- No assumptions made without code evidence

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: Observation 异常？ → If I missed a requirement in Observation, would the implementation be incomplete?
- Q2: 解读正确？ → Is there a referenced file in the design that I didn't read?
- Q3: 还有遗漏？ → Would project manager think I understood all requirements based on Observation?

You must complete at least **2 loops** before you can PASS this step.
