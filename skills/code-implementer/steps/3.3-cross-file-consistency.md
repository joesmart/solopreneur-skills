# Step 3.3: Cross-File Consistency Check

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

Single-file compliance doesn't guarantee overall consistency. Check:
- **Naming consistency**: same concept uses same name across all files
- **Interface contracts**: Controller input/output matches BizService method signatures
- **Data flow completeness**: from entry to exit, data type conversions are correct, nothing is missing
- **Exception handling**: exceptions are caught and handled at the correct layer

If issues found, fix and re-verify.

## Checklist

Use these items in your **[CHECK]** stage:

- Naming consistency verified across all files
- Interface contracts verified between layers
- Data flow traced end-to-end
- Data type conversions verified
- Exception handling verified at each layer
- All cross-file integration points checked
- Issues fixed and re-verified

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: Observation 异常？ → If I trace one complete request flow in Observation, would I find inconsistencies?
- Q2: 解读正确？ → Is there a data type mismatch between layers in Observation?
- Q3: 还有遗漏？ → Would integration tester find issues in cross-file integration from Observation?

You must complete at least **2 loops** before you can PASS this step.
