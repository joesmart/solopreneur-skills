# Step 2.4: Code Correctness Verification

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

Spec compliance is formal correctness — also verify logical correctness:
- Use getDiagnostics to check syntax errors, type errors
- Check boundary conditions: null handling, exception handling, concurrency safety
- Check upstream/downstream code integration correctness

**Loop Rule**: If issues found, loop back to Step 2.2 to fix, then re-verify.

## Checklist

Use these items in your **[CHECK]** stage:

- getDiagnostics run and no errors found
- Null handling verified for all inputs
- Exception handling appropriate for layer
- Concurrency safety considered
- Integration with upstream/downstream code verified
- No edge cases unhandled

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: Observation 异常？ → What input would cause this code to fail based on Observation?
- Q2: 解读正确？ → Is there an edge case I didn't consider in Observation?
- Q3: 还有遗漏？ → Would QA engineer find bugs in this code from Observation?

## Minimum Loops

You must complete at least **1 loop** before you can PASS this step.
