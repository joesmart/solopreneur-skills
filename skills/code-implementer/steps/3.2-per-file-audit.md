# Step 3.2: Per-File Full Spec Audit

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

For each file from Step 3.1:
- Re-read the file's complete code
- Re-load all applicable specs from `../../specs/` (per `_shared/spec-loader-rules.md`)
- Check every MUST / MUST NOT item against the code
- Output audit results:

```
### [File Path]
Applicable specs: [spec1], [spec2]
✅ [MUST] Item 1: Compliant
❌ [MUST] Item 2: Non-compliant - [specific issue]
```

**If ANY non-compliant item**: fix immediately, then re-audit that file until 100% pass.

## Checklist

Use these items in your **[CHECK]** stage:

- Every file from inventory audited
- File code actually re-read (not just Phase 2 memory)
- All applicable specs re-loaded
- Every MUST/MUST NOT item checked
- Audit results documented for each file
- Non-compliant items fixed and re-audited
- 100% compliance achieved for all files

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: Observation 异常？ → Is this a genuine re-audit or just repeating Phase 2 results based on Observation?
- Q2: 解读正确？ → Did I actually re-read the code or rely on memory in Observation?
- Q3: 还有遗漏？ → Would code reviewer think I genuinely re-checked each item from Observation?

## Minimum Loops

You must complete at least **2 loops** before you can PASS this step.
