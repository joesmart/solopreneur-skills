# Step 2.1: Load Applicable Code Specs

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

Before writing any code, determine which specs apply to the current implementation step.

Run the spec resolver script:

```bash
SHARED="$(dirname "${CLAUDE_SKILL_DIR}")/_shared/scripts"
bash "$SHARED/resolve-specs.sh" <target_file_path>
```

Or consult [spec-loader-rules.md](../../_shared/spec-loader-rules.md) for the task-type mapping:

| 任务类型 | 必须加载的规范 |
|---------|---------------|
| 编写代码 | `clean-function`, `clean-naming`, `error-handling` |
| 数据库相关 | `data-integrity` |
| 安全相关 | `security` |

**Read each required spec file** from `../../specs/{spec-id}/spec.md`. Extract all MUST / MUST NOT items as the coding constraint checklist.

## Checklist

Use these items in your **[CHECK]** stage:

- All applicable specs identified based on task type
- Every required spec file actually read
- MUST/MUST NOT checklist extracted from each spec
- Checklist is complete and specific
- No spec that should apply was missed

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: Observation 异常？ → If I missed a spec in Observation, would my code violate standards?
- Q2: 解读正确？ → Is there a spec that applies but I didn't load based on Observation?
- Q3: 还有遗漏？ → Would code reviewer think I loaded all relevant specs based on Observation?

## Minimum Loops

You must complete at least **1 loop** before you can PASS this step.
