# Step 2.2: Implement Code Strictly Per Spec

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

Write code under the spec constraints:
- If modifying an existing file, **read the existing code first** to understand context
- If the design contains specific code examples or pseudocode, **implement strictly per the design** — do not improvise
- Cross-check against the spec constraint checklist while coding
- Follow existing code's naming conventions, code style, and architecture patterns
- Add clear comments explaining intent for each method

## Checklist

Use these items in your **[CHECK]** stage:

- Existing code read and understood before modification
- Code follows design exactly (no improvisation)
- Every spec constraint checked during coding
- Code matches existing codebase's style
- Naming conventions consistent with existing code
- Comments added for method intent
- No spec constraint violated

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: Observation 异常？ → If I review my code line by line in Observation, would I find spec violations?
- Q2: 解读正确？ → Did I improvise anywhere instead of following the design based on Observation?
- Q3: 还有遗漏？ → Would code reviewer think this matches the existing codebase style from Observation?
