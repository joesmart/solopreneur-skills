# Step 2.3: Spec Compliance Check (MUST NOT SKIP)

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

After code is written, verify 100% compliance with all loaded specs. Output check results in this format:

```
Spec Compliance Check:
✅ [Spec item 1]: Compliant - [brief explanation]
✅ [Spec item 2]: Compliant - [brief explanation]
❌ [Spec item 3]: Non-compliant - [specific issue description]
```

- **If ANY item is non-compliant**: fix the code immediately, then re-run 2.3 until 100% compliant
- **If ALL items pass**: mark this step complete, proceed to next step

**CRITICAL RULE**: MUST NOT proceed to next implementation step until 100% spec compliance achieved.

## Checklist

Use these items in your **[CHECK]** stage:

- Every MUST/MUST NOT item from loaded specs checked
- Each check has concrete evidence (not "should be fine")
- Non-compliant items clearly documented with specific issues
- If non-compliant, code fixed and re-checked
- 100% compliance achieved before proceeding

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: Observation 异常？ → Did I rubber-stamp any item without careful verification in Observation?
- Q2: 解读正确？ → Is there a non-obvious spec item I skipped based on Observation?
- Q3: 还有遗漏？ → Would code reviewer think I genuinely checked every item from Observation?
