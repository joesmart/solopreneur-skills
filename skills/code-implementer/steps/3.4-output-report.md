# Step 3.4: Output Final Compliance Report & Change List

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

After all verification passes, output the final report:

```

### Audit Result: ✅ All Passed / ❌ Issues Found

| File | Applicable Specs | Status |
|------|-----------------|--------|
| [file path] | [spec list] | ✅ Passed |

## Change List

### New Files
- [file path]: [one-sentence description]

### Modified Files
- [file path]: [change summary]

### SQL Changes (if any)

```

## Checklist

Use these items in your **[CHECK]** stage:

- Report format follows template exactly
- All audited files listed in report
- Audit result clearly stated (All Passed / Issues Found)
- Change list includes all new files
- Change list includes all modified files
- SQL changes listed if any
- Report is complete and accurate

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: Observation 异常？ → If developer reads this report from Observation, would they understand what was implemented?
- Q2: 解读正确？ → Is any file change missing from the change list in Observation?
- Q3: 还有遗漏？ → Would project manager think this report from Observation is complete and professional?
