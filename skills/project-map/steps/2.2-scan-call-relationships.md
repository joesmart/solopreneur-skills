# Step 2.2: Scan Call Relationships

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

Based on Phase 1 detection result:

**If Microservices - Search for inter-service communication:**

**HTTP sync calls:**
- Java: `RestTemplate`, `FeignClient`, `HttpClient`, `WebClient`
- PHP: `Guzzle`, `Http::` facade, `curl`
- Node: `axios`, `fetch`, `http.request`
- Config files: `http://xxx-service` URLs

**Message queue async calls:**
- RabbitMQ: `@RabbitListener`, `RabbitTemplate`
- Kafka: `@KafkaListener`, `KafkaTemplate`
- Redis Pub/Sub, other MQ

**RPC calls:** gRPC, Dubbo, etc.

For each call record: caller → callee, method (HTTP/MQ/RPC), purpose.

**If Monolith - Search for cross-module calls:**
- `import`, `require`, `use` statements referencing other modules
- Service layer cross-module injection and calls

For each call record: caller → callee, purpose.

## Checklist

Use these items in your **[CHECK]** stage:

- All communication patterns searched (HTTP/MQ/RPC for microservices)
- Each call relationship documented with caller, callee, method, purpose
- Config-based service URLs found (if applicable)
- Cross-module dependencies complete (for monolith)
- No protocol/pattern missed

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: Observation 异常? → Are there calls that don't fit expected patterns?
- Q2: 解读正确? → Did I correctly identify the purpose of each call?
- Q3: 还有遗漏? → Any protocol missed (e.g., WebSocket, event bus)? Indirect dependencies (shared DB tables, events)?
