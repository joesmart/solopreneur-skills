# Step 1.1: Scan Root Directory

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

List all files and directories at project root. Check for:

**Microservices indicators:**
- `services/`, `apps/`, `packages/` directories containing multiple sub-projects
- Multiple independent `pom.xml` / `composer.json` / `package.json`
- `docker-compose.yml` defining multiple services

**Monolith indicators:**
- Single `src/` or `app/` directory
- Single build config file (`pom.xml` / `composer.json` / `package.json`)

## Checklist

Use these items in your **[CHECK]** stage:

- Root directory listed completely
- Microservices indicators identified (if present)
- Monolith indicators checked (if present)
- Build config files located
- Project type determined with confidence

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: Observation 异常? → Are there conflicting signals (e.g., both services/ and single src/)?
- Q2: 解读正确? → Could this be a monorepo that looks like microservices but isn't?
- Q3: 还有遗漏? → Did I check for hidden directories or non-standard structures?

You must complete at least **2 loops** before you can PASS this step.
