# Step 1.2: Identify Tech Stack

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

Check build/dependency config files to identify tech stack:

| Language | Config File | Framework Check |
|----------|------------|-----------------|
| Java | `pom.xml` / `build.gradle` | `spring-boot` |
| PHP | `composer.json` | `laravel/framework` |
| Node.js | `package.json` | `express` / `nestjs` / `koa` |
| Python | `requirements.txt` / `setup.py` | `django` / `flask` / `fastapi` |
| Go | `go.mod` | framework dependencies |

For microservices projects, identify tech stack per sub-service.

## Checklist

Use these items in your **[CHECK]** stage:

- All dependency files located and read
- Primary language identified with evidence
- Framework identified with version
- For microservices: tech stack per service documented
- No dependency file left unchecked

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: Observation 异常? → Are there multiple languages/frameworks in use? Is this expected?
- Q2: 解读正确? → Did I correctly identify the framework from dependencies?
- Q3: 还有遗漏? → Are there sub-services whose tech stack is still unknown?

## Minimum Loops

You must complete at least **2 loops** before you can PASS this step.
