# Step 2.1: Scan Service/Module Inventory

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

Based on Phase 1 detection result:

**If Microservices:**
- List all service directories
- For each service record: name, responsibility, tech stack, code path
- Check for API Gateway, shared libraries, infrastructure components

**If Monolith:**
- Identify all business modules by layered architecture
- Java: by package name (`com.company.project.user`, `com.company.project.order`)
- PHP/Laravel: by Controller/Model/Service directories
- Node: by routes/controllers/services directories
- For each module record: name, responsibility, code path

## Checklist

Use these items in your **[CHECK]** stage:

- Complete service/module list generated
- Each item has name, responsibility, tech stack (if microservices), code path
- Infrastructure components identified (if microservices)
- No hidden services/modules overlooked
- Responsibilities clearly annotated

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: Observation 异常? → Are there services/modules whose responsibility is unclear?
- Q2: 解读正确? → Did I correctly identify all business domains?
- Q3: 还有遗漏? → Any hidden services (e.g., shared libs acting as services, sidecar containers)?

## Minimum Loops

You must complete at least **2 loops** before you can PASS this step.
