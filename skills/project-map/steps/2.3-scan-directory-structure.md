# Step 2.3: Scan Directory Structure

**Apply the ReAct protocol from SKILL.md to execute this step.**

## Action

Based on Phase 1 detection result:

**If Microservices:**
- For each service, list 2-3 levels of directory structure
- Annotate each directory's responsibility (controller, service, dao, model, etc.)

**If Monolith:**
- List 2-3 levels of project directory structure
- Annotate each directory/layer's responsibility
- Identify architecture pattern (MVC / Controller-Service-DAO / DDD, etc.)

## Checklist

Use these items in your **[CHECK]** stage:

- Directory structure listed to 2-3 levels depth
- Each directory annotated with responsibility
- Architecture pattern identified (for monolith)
- No directory with unclear purpose
- Structure accurately represents code organization

## Challenges

Use these questions in your **[CHALLENGE]** stage:

- Q1: Observation 异常? → Are there directories whose purpose doesn't match their name?
- Q2: 解读正确? → Did I correctly identify the architecture pattern?
- Q3: 还有遗漏? → Any layer missing from the structure? Should I check code inside unclear directories?

## Minimum Loops

You must complete at least **2 loops** before you can PASS this step.
